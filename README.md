# Block List View Class（Chrome拡張機能版）

WordPressブロックエディターのリストビューを拡張するChrome拡張機能です。
サイトごとのプラグインインストール不要で、自分が編集する全WordPressサイトで使えます。

## 機能

- **ブロック名のインライン編集**
  リストビューのブロック名をダブルクリックで編集。Enterで確定、Escapeでキャンセル。
  値は`metadata.name`としてブロック属性に保存されます（WordPress 6.5以降）。

- **追加CSSクラスの直接入力**
  リストビュー各行にクラス入力欄を表示。入力は400msのデバウンス後に`className`属性へ反映。
  サイドバーの「追加CSSクラス」と双方向で同期します。

- **複数ブロックの一括編集**
  各行のチェックボックスで複数選択すると、リストビュー上部に一括編集パネルを表示します（サイドバー構造が見つからない場合は画面左下に固定表示）。
  - 名前の一括変更（「連番」チェックで`-1`, `-2`…を付加）
  - クラスの一括編集（追加・置換・削除）
  - 選択解除

- **右クリックメニュー（コピー機能）**
  リストビューの行を右クリックすると表示。
  - DOMツリーをコピー（HTML）：エディター専用の属性・クラスを除外したクリーンなHTML
  - CSSをコピー：純粋CSS・SCSSネスト・SCSSフラット・Tailwindクラス一覧の4形式
  - セレクターは「追加CSSクラス→wp-block-\*クラス→タグ名」の優先順で決定

- **ヒント表示**
  本拡張機能が追加したUI（チェックボックス・クラス入力欄・リストビュー行など）にマウスオンして少し待つと、吹き出しで操作ヒントを表示します。

## インストール手順

1. Chromeで`chrome://extensions`を開く
2. 右上の「デベロッパーモード」をONにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. このフォルダー（`manifest.json`があるディレクトリ）を選択

更新時は`chrome://extensions`で本拡張機能の「更新」ボタン（リロードアイコン）を押してください。

## 対応ページ

以下のURLパターンで自動的に有効になります。サブディレクトリ設置のWordPressにも対応しています。

- `*/wp-admin/post.php*`（投稿・固定ページの編集）
- `*/wp-admin/post-new.php*`（新規作成）
- `*/wp-admin/site-editor.php*`（サイトエディター）

`wp.data`（ブロックエディター）が存在しないページでは何も行いません。
Local by Flywheelのローカルサイト（`http://xxx.local/wp-admin/`）でも動作します。

## ファイル構成

```text
block-list-view-class-chrome/
├── manifest.json        # Manifest V3定義
├── content/
│   ├── main.js          # 本体ロジック（MAIN worldで実行）
│   └── style.css        # 注入スタイル
├── icons/               # プレースホルダーアイコン
└── reference/           # 元プラグインの参考実装
```

## 技術メモ

- コンテンツスクリプトは`"world": "MAIN"`で注入し、ページ側のグローバル`wp.data`（`select`/`dispatch`/`subscribe`）を直接操作します（Chrome 111以降が必要）。
- エディターキャンバスが`iframe[name="editor-canvas"]`内にある場合も、同一オリジンのため`contentDocument`経由でアクセスします。
- リストビュー行への注入は`data-blvc-injected`属性でガードし、リストビューの開閉で重複しないようにしています。
- 入力欄ではkeydown/copy/cut/pasteの伝播を停止し、WordPressのキーボードショートカットとの衝突を防いでいます。
